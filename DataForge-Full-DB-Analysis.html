05 — Core Features

Everything you can do

Every feature gives you instant cost-awareness as your architecture evolves.

⬡

Visual Data Modelling

Drag-and-drop canvas. Central model (User) with record ratios — "1 user → 50 orders, 200 events". Schema adapts per DB type.

⟁

28-Database Support

SQL, NoSQL, Graph, Vector, In-Memory, Search, Time-Series. Switch DB and see how schema, sharding, and cost change instantly.

◈

Sharding Planner

Per-DB shard strategies: hash/range keys (Mongo), Citus (PG), vnodes (Cassandra), hash slots (Redis), partitions (DynamoDB).

◇

Cache Layer Designer

Redis, Valkey, Memcached, or Dragonfly. Define TTLs, eviction policies, memory budgets. Memory cost projections at scale.

↗

Growth Simulator

Slide from 1K → 10M users. Watch records, storage, IOPS, and cost update live. Compare multiple DBs side-by-side.

$

Live Cost Engine

Real pricing: Atlas, RDS, AuraDB, Pinecone, ElastiCache, Elastic Cloud and more. Compute + storage + backup + network.

⊞

Schema Export

Prisma, Mongoose, Cypher, raw DDL, DynamoDB JSON, Elasticsearch mappings. One-click scaffold for your codebase.

⚡

MCP Server

Run DataForge as an MCP server. Claude, Cursor, or any AI assistant can query models, estimate costs, and suggest optimizations.

07 — Execution Phases

From zero to launch

Phased approach — ship early, iterate fast. Each phase delivers a usable product.

01

Foundation

Weeks 1–3
  • Monorepo setup (Turborepo)
  • Core Model Graph Engine — entities, fields, relationships
  • Record ratio calculator (central → sub-models)
  • React canvas with entity boxes & connections
  • DB adapter interface + first 4 adapters (PG, Mongo, Neo4j, Pinecone)
  • Project persistence (SQLite/JSON)
02

Cost Engine

Weeks 4–6
  • Cost model abstractions per DB category
  • SQL adapters: PG/RDS, MySQL, Oracle, SQL Server, MariaDB, CockroachDB
  • NoSQL adapters: MongoDB Atlas, DynamoDB, Cassandra, Firebase
  • Cost dashboard with growth slider (1K → 10M)
  • Real-time recalculation on model changes
03

All DB Adapters

Weeks 7–9
  • In-Memory: Redis, Valkey, Memcached, Dragonfly pricing
  • Graph: Neo4j AuraDB, Neptune, ArangoDB, Dgraph
  • Vector: Pinecone, Weaviate, Milvus/Zilliz, Qdrant, pgvector, Chroma
  • Search: Elasticsearch, OpenSearch
  • Time-Series: InfluxDB, TimescaleDB
  • Cross-DB comparison view
04

Sharding & Caching

Weeks 10–12
  • Per-DB shard configuration UI
  • Shard distribution simulator & hot-spot detection
  • Cache layer planner (Redis/Valkey/Memcached/Dragonfly)
  • Cache memory cost projections
  • Schema export (Prisma, Mongoose, Cypher, DDL, mappings)
05

MCP Server

Weeks 13–15
  • MCP server with stdio + SSE transports
  • 8+ tools with JSON Schema definitions
  • Test with Claude Desktop & Cursor
  • Templates: SaaS, E-commerce, Social, IoT
  • UI polish — animations, themes, responsive
06

Open Source Launch

Week 16
  • GitHub repo — README, CONTRIBUTING, MIT License
  • npm package (npx dataforge)
  • Docker image (docker run dataforge)
  • MCP registry submission
  • Launch: HN, Product Hunt, Reddit, X
